# Security Policy

## Supported Versions

The latest release on the default branch is the supported version for security fixes.

| Version | Supported |
| ------- | --------- |
| 2.x     | :white_check_mark: |
| < 2.0   | :x: |

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

1. Open a private security advisory in GitHub, if available for the repository.
2. If private reporting is unavailable, contact the maintainer through the repository owner profile and include:
   - affected version
   - impact summary
   - reproduction steps or proof of concept
   - suggested remediation, if known
3. Allow reasonable time for verification and remediation before public disclosure.

Security reports should avoid including production secrets, API keys, or customer data.
